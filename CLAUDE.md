## О проекте

Dockform Portable — портативное Windows-приложение для автоматического формирования юридических документов из Word-шаблонов. Сотрудник запускает `.exe` с флешки или из любой папки (без установки), выбирает шаблон, заполняет форму, получает готовый `.docx`. Работает офлайн. Шаблоны и сама программа обновляются с GitHub при наличии интернета. Есть удалённая блокировка (kill switch).

Это переписывание старого проекта (n8n + Flask + Google Forms + Telegram → единое локальное приложение). Старая логика n8n не переиспользуется, переезжает в Word-шаблоны как условные блоки `{#field}...{/field}`.

## Как запускать

```bash
# Разработка (открывает десктоп-окно с hot reload)
npm run tauri dev

# Сборка релизного бинарника
npm run tauri build
```

Тестирование режимов запуска:
```bash
# В режиме разработки аргументы прокидываются через три уровня (npm → tauri → cargo → бинарь),
# поэтому нужно три «--» подряд.
npm run tauri dev -- -- -- --mode=pretenzii
npm run tauri dev -- -- -- --mode=documentation
```

## Где что лежит (карта для кода)

- **Tauri command (новый канал между фронтом и Rust)** → `src-tauri/src/commands/{домен}.rs`, регистрация в `main.rs`. На фронте — обёртка в `src/lib/tauri.ts`.
- **Новый тип поля формы** → расширить `FieldType` и `FieldConfig` в `src/types/`, добавить ветку рендера в `DynamicForm`, обновить `TZ.md` секция 6.2.
- **Новый экран** → `src/pages/`, добавить навигацию в `App.tsx`.
- **Чистая логика без UI** → `src/lib/`. Не клади бизнес-логику в компоненты.
- **Файловые операции** → всегда через Rust commands. JS НЕ должен напрямую дергать `fs` или `path` — это ломает портативность и безопасность.
- **Сетевые запросы** → в Rust (`commands/network.rs`). JS дергает Tauri command, не fetch напрямую.
- **Новый калькулятор** → `src/lib/calculators/<id>.ts` (экспорт `CalculatorDef`), регистрация в `src/lib/calculators/index.ts`, юнит-тест рядом (`<id>.test.ts`). YAML-шаблоны ссылаются по `id`. Универсальные и специфичные — в одном реестре, без иерархии.
- **Развилка «или-или» в форме (напр. срок днями ИЛИ датой)** → radio + поля под взаимоисключающими `visible_if`. Если ветки используют разные калькуляторы, пишущие в одни и те же переменные `.docx`, пометить оба блока одинаковым `variant_group:` — иначе валидатор `template-validator.ts` посчитает это коллизией выходов. Пример: `delivery_overdue` + `overdue_from_deadline` в `44_аук_единовр_пост_претензия.yaml`.
- **Сумма прописью у нового шаблона** → ставим `text_output: <имя_прописью>` у `type: number` в YAML и используем `{<имя_прописью>}` в `.docx`. Кода писать не нужно — функция `src/lib/amount-to-words.ts` уже подключена.
- **Новый шаблон претензий** → `templates/pretenzii/<имя>.docx` + `.yaml` с блоком `tags:` (оси из `_wizard.yaml`). При необходимости расширить варианты ответов в `templates/pretenzii/_wizard.yaml`. Логика анкеты — `src/lib/wizard-evaluator.ts`, валидатор — `src/lib/wizard-validator.ts`, экран — `src/pages/WizardPage.tsx`.
- **Изменение структуры анкеты претензий** → правки в `templates/pretenzii/_wizard.yaml`; добавление новой оси требует дописать тег во все существующие шаблоны претензий (иначе валидатор заблокирует старт).
- **Несколько документов из одной формы** → блок `extra_templates:` в YAML основного шаблона (доп. `.docx` в той же папке). Код генератора расширять не нужно — связку собирает `generateBundle` в `src/lib/generator.ts`.
- **Сборка portable / выпуск новой версии** → `docs/release.md` (полный workflow на Windows: bump версии → `npm run tauri build` → собрать папку `dockform/` с `templates/`, `dictionaries/`, `cache/`, `output/`, `README.txt` → упаковать в zip → раздать пилотам). **.lnk в zip НЕ кладём** — их создаст сам `dockform.exe` при первом запуске у пилота (см. ниже).
- **Ярлыки `Претензионная.lnk` / `Документация.lnk` для папки портабла** → автосоздаются Rust-кодом при старте release-сборки на Windows (`src-tauri/src/commands/shortcuts.rs`, crate `mslnk`). Перезаписываются на каждом старте — путь всегда совпадает с текущим расположением .exe. Раньше создавались скриптом `scripts/make_shortcuts.ps1`, который зашивал абсолютный путь со сборочного ПК и ломал ярлыки при копировании на чужой ПК; скрипт оставлен в репо как ручной фолбэк, но в основном workflow не используется. README для конечного пользователя — `docs/portable-README.txt`, копируется в папку как `README.txt`.

## Конвенции

- **Именование файлов:** kebab-case (`form-evaluator.ts`, `templates-list.tsx`).
- **Именование компонентов:** PascalCase (`<DynamicForm>`, `<TemplateCard>`).
- **Именование Rust-функций:** snake_case (`list_templates`, `save_draft`).
- **Имена переменных в YAML и Word-шаблонах:** на русском, snake_case_or_lowercase (`нмцк_цифра`, `применяется_запрет`). Это сохраняется из старого проекта для совместимости.
- **TypeScript:** строгий режим (`strict: true`), `any` — только в исключительных случаях с комментарием.
- **Импорты:** алиас `@/` указывает на `src/`. Использовать его, не относительные `../../`.
- **Стилизация:** только Tailwind utility-классы, без отдельных CSS-файлов кроме `index.css`. Если нужен сложный кастомный стиль — `cn()` хелпер из `lib/utils.ts` (приходит с shadcn).
- **Комментарии в коде:** на русском, кратко, только там где «почему», а не «что».
- **Сообщения коммитов:** на русском, формат `Этап N: краткое описание`. Для мелких правок внутри этапа — `Этап N: фикс {что}`.
- **Даты в выходных документах** — строго `дд.мм.гггг` (`15.02.2026`). Не ISO, не «February…». Калькуляторы возвращают даты уже отформатированные через хелпер; в `.docx` юрист пишет фиксированные даты в этом же формате.
- **Суммы и десятичные числа в выходных документах** — строго с запятой (`1 234,56`). Точка как десятичный недопустима. Калькуляторы с денежными выходами форматируют результат строкой через `toLocaleString("ru-RU", …)`, а не отдают сырой `Number` (иначе docxtemplater подставит `1234.56`).

## Критические правила (НЕ нарушать)

1. **Все файловые пути — относительно `current_exe()` parent.** Никогда не используй `home_dir()`, `app_data_dir()`, `temp_dir()`. Это ломает портативность. Папки `templates/`, `dictionaries/`, `cache/`, `output/` всегда лежат рядом с `.exe`.
2. **Никаких сетевых вызовов из JS.** Все запросы — через Rust commands с таймаутом (не больше 5 секунд). Иначе при обрыве сети UI зависает.
3. **Старт приложения должен укладываться в 5 секунд при отсутствии интернета.** Все сетевые проверки (kill switch, manifest, updater) — асинхронные, неблокирующие. UI открывается сразу с локальным состоянием, обновления подтягиваются в фоне.
4. **Никаких сторонних API-ключей, токенов, секретов в коде.** В этом приложении их быть не должно вообще. Если что-то такое нужно — стоп, обсуди с пользователем.
5. **shadcn-компоненты не редактировать.** Они добавляются через `npx shadcn@latest add` и считаются «зашитыми». Если нужна модификация — оборачивай в свой компонент.
6. **YAML-конфиги шаблонов — единственный источник правды о структуре формы.** Не дублируй информацию о полях в TypeScript-коде. Форма строится динамически.
7. **Не вводить новые зависимости без необходимости.** Если задачу решает уже установленный пакет — использовать его. Каждый новый npm/cargo-пакет = +вес дистрибутива и +поверхность атаки.
8. **Не работать вне текущего этапа** (см. секцию «Текущий этап»). Если возникает соблазн «заодно поправить вот это» из следующего этапа — остановиться, спросить пользователя.

## Word-шаблоны — синтаксис docxtemplater

**Разделители — одинарные `{` и `}`** (НЕ `{{ }}`, как было в docxtpl старого проекта). При миграции старых шаблонов — заменить.

Простая подстановка:
```
ИКЗ закупки: {ikz}
НМЦК: {нмцк_цифра} ({нмцк_текст}) рублей
```

Условный блок (показывается, если поле truthy):
```
{#применяется_запрет}
Запрет на допуск товаров иностранного производства... согласно п. {пункт_тз_запрет} ТЗ.
{/применяется_запрет}
```

Инверсный блок (показывается, если поле falsy / отсутствует / 'Нет'):
```
{^применяется_запрет}
Запрет не применяется.
{/применяется_запрет}
```

Цикл по массиву:
```
{#товары}
- {наименование}: {количество} шт.
{/товары}
```

**Truthy в docxtemplater:** строки `'Да'`, любое непустое значение → блок показывается. Строки `'Нет'`, пустота, `false` → не показывается. Это встроенное поведение, специальной настройки не требует.

## Частые ошибки и подводные камни

- **Плейсхолдер в `.docx` разбит на несколько runs.** Когда юрист правит документ в Word, плейсхолдер `{переменная}` может оказаться разбитым на несколько XML-элементов из-за форматирования. docxtemplater тогда не видит его. Решение: открыть файл в Word, выделить плейсхолдер и применить `Ctrl+Space` (сброс форматирования) или ввести его заново. Скрипт `extract_variables.py` помогает находить такие случаи.
- **Path с пробелами в Windows.** Если `.exe` лежит в папке вроде `C:\Users\Имя Пользователя\Desktop\dockform\`, путь содержит пробел. Все Rust-операции с путями должны использовать `PathBuf`, не строки. На фронте — не строить пути конкатенацией строк, всегда через Tauri.
- **Tauri 2 ≠ Tauri 1.** Многие гайды в интернете ещё про Tauri 1. API сильно отличается (особенно plugin система, capabilities, command registration). Сверяться с актуальной документацией Tauri 2.
- **WebView2 на старых Win10.** На Windows 10 без свежих обновлений может не быть WebView2. Bundle setting `webviewInstallMode: "downloadBootstrapper"` решает это автоматически при первом запуске.
- **Кодировка YAML.** Файлы YAML и JSON всегда в UTF-8 без BOM. Если кириллица в `display`-полях рендерится как `?` — проверить, что редактор сохранил в UTF-8.
- **Блокировка GitHub Raw.** Иногда корпоративные сети блокируют `raw.githubusercontent.com`. Если у конкретного пользователя обновления не приходят — это первая проверка. Приложение должно нормально работать в этом случае, просто без обновлений.
