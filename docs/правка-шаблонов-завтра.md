# Правка шаблонов через GitHub-обновление + текущая задача

Конспект для возобновления работы. Включает:
1. Постоянный workflow «путь С» — как любая правка шаблона доезжает до пилота автоматически.
2. Текущая задача: условный блок для акта приёма на ответственное хранение в 5 шаблонах претензий.

---

## 1. Workflow правки шаблонов (путь С — автообновление)

Цикл:

1. **Поправить файл локально:**
   - `.docx` — в Word (правый клик → «Открыть с помощью» → Word).
   - `.yaml` — в любом текстовом редакторе.
2. **Протестировать локально:**
   ```bash
   npm run tauri dev
   ```
   Пройти кейс, который правил, убедиться, что готовый `.docx` корректный.
3. **Пересчитать манифест:**
   ```bash
   python3 scripts/build_manifest.py
   ```
   Скрипт пересчитает SHA-256 всех файлов в `templates/` и `dictionaries/`, обновит `manifest.json`. Папка `old/` и файлы `.md` игнорируются (не уезжают к пилотам).
4. **Коммит и пуш** (через Claude или сам):
   ```bash
   git add templates/<твой файл> manifest.json
   git commit -m "Шаблон <имя>: <что поправил>"
   git push origin main
   ```
5. **Пилот ничего не делает** — при следующем запуске `dockform.exe` сам сходит на `raw.githubusercontent.com`, увидит изменившийся хеш, скачает новый файл поверх старого.

### Что важно

- **Если пилот в момент обновления уже в форме** — текущая открытая форма не перерендерится. Новый шаблон подхватится при следующем открытии того же шаблона из списка.
- **Если у пилота нет интернета** — он работает со старой версией, обновление подтянется, когда сеть появится.
- **Проверка доступа у пилота:** открыть в браузере на пилотной машине `https://raw.githubusercontent.com/hermitage-cyber/dockform/main/manifest.json`. Должен показать JSON. Если нет — корпоративный фильтр / антивирус режет домен.

## 2. Текущая задача: условный блок «акт отв. хранения»

### Проблема

В пяти шаблонах претензий есть фраза:
```
{номер_дата_акта_отв_хранения} подписан ФГБУК «Государственный Эрмитаж» (далее - Заказчик) {дата_акта_отв_хранения}, документ о приемке {номер_дата_документа_приемки}
```

Поле `номер_дата_акта_отв_хранения` опциональное. Если его не заполнить — в готовом `.docx` остаётся «висячий» текст про подписание и лишняя запятая.

### Решение

Обернуть всю упоминание акта в условный блок docxtemplater. Запятая и пробел перед «документ» — **внутри** блока.

Меняем на:
```
{#номер_дата_акта_отв_хранения}{номер_дата_акта_отв_хранения} подписан ФГБУК «Государственный Эрмитаж» (далее - Заказчик) {дата_акта_отв_хранения}, {/номер_дата_акта_отв_хранения}документ о приемке {номер_дата_документа_приемки}
```

Логика:
- Поле пустое → docxtemplater считает falsy → блок целиком убирается. Остаётся «… документ о приемке № … от …».
- Поле заполнено → блок показывается, всё подставляется как раньше с запятой.

### Файлы для правки

Все в `templates/pretenzii/`:
- `44_аук_единовр_пост_претензия.docx`
- `44_аук_1част_пост_претензия.docx`
- `44_аук_2част_пост_претензия.docx`
- `44_аук_3част_пост_претензия.docx`
- `44_аук_4част_пост_претензия.docx`

### Как править в Word без подводных камней

Главная ловушка — **плейсхолдер, разбитый на runs**. Когда Word из-за форматирования режет `{поле}` на несколько XML-кусков, docxtemplater его не видит. Алгоритм против этого:

1. Открыть шаблон в Word.
2. `Ctrl+F` → найти `подписан ФГБУК`.
3. **Выделить всю фразу целиком** — от `{номер_дата_акта_отв_хранения}` до запятой и пробела перед «документ».
4. Нажать **`Ctrl+Space`** на выделении — сбросит локальное форматирование, runs склеятся.
5. Удалить выделенное и **набрать новый текст руками** (вставка из буфера может прийти со скрытым форматированием — надёжнее набрать). Условные маркеры `{#...}` и `{/...}` не печатаются в финальном документе, это служебные конструкции docxtemplater.
6. Сохранить файл, **не** меняя имя.

Если в каком-то шаблоне фраза отличается от приведённой (другая редакция вокруг плейсхолдеров) — сначала скинуть Claude точный текст, он подскажет границы блока для этого случая.

### Проверка локально

```bash
npm run tauri dev
```

Прогнать оба кейса:
1. **С актом** — заполнить поле `номер_дата_акта_отв_хранения` (например, «б/№ от 27.03.2026»). В готовом `.docx` всё как раньше, с запятой перед «документ».
2. **Без акта** — оставить поле пустым. Упоминания акта быть не должно, между предыдущим текстом и «документ о приёмке» один пробел, запятой нет.

Если в кейсе «без акта» остался лишний пробел / двойной пробел / запятая — границы блока в Word захватили не то, поправить.

### Финал

Когда все 5 файлов поправлены и оба кейса прошли локально:
```bash
python3 scripts/build_manifest.py
```
Дальше — Claude делает коммит и пуш, пилот при следующем запуске получает правки.
